【Java】synchronized和lock有哪些区别？
一、实现原理
synchronized对象底层通过monitorenter、monitorexit实现
synchronized作用于方法，通过ACC_SYNCHRONIZED指令完成
是悲观并发策略
lock是基于AQS是实现的同步非阻塞，采用乐观策略
二、使用方法
synchronized是关键字，作用与代码块、方法
lock是接口
三、在异常处理上
synchronized在发生异常时会自动释放
lock要在finally块中释放
四、锁的优缺点
synchronized不能让等待锁的线程中断
synchronized不饿能知道锁的获取状态
lock能让等待锁的线程中断，且能知道有没有获取锁
lock可以绑定多个条件
