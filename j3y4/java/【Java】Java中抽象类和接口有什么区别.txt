Java中抽象类和接口有什么区别？
1.抽象类可以包含抽象方法和具体方法，而接口只能包含抽象方法和常量
抽象方法是没有实现的方法，必须在子类中被实现
2.类只能继承一个抽象类，但可以实现多个接口。
3.抽象类可以有构造方法，而接口没有构造方法。
4.抽象类通常用于代码复用和扩展性设计，接口通常用于实现多态和解耦。
抽象类可以在子类中定义一些公共的属性或方法，
这样可以避免代码重复，提高代码的可读性和可维护性。
抽象类可以用于实现模板方法模式，
即定义一个算法的骨架，将一些步骤延迟到子类中实现，从而实现算法的复用和扩展。
例如：
AbstractController：这是Spring MVC中控制器的抽象类，它提供了一些通用的方法和属性
例如视图解析器、请求处理等。
AbstractFactoryBean：这是Spring框架中用于创建Bean的抽象类，
它提供了一些通用的方法和属性，例如Bean名称、Bean类型等。
