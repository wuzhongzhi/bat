GC如何判断对象可以被回收？
判断对象回收算法：
1.引用计数算法：
对象中添加引用计数器，有一个加1，引用失效减1，为零就回收
引用计数算法存在循环引用问题
2.可达性分析算法：
从GC ROOTS起点开始向下搜索
搜索走过的路径成为引用链
如果对象和gc roots没有引用链，就是不可达的
Gc roots有以下对象：
栈帧中引用对象（堆栈中使用到的参数、局部变量等）
方法区静态属性引用的对象（引用类型静态变量）
方法区中常量引用的对象（String Table）
native方法引用的对象
虚拟机内部的引用（类加载器、nullpointerexception等）
同步锁持有的
本地代码缓存等
对象回收的区域：
垃圾回收不会发生在永久代，但是，永久代满了或者超过临界值、会触发
对象回收的算法
标记清除（先标记然后清除）
标记整理（先标记、然后将存活对象像一端移动）
标记复制（先标记、然后复制到另一半内存中）
